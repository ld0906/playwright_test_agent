EP 05/审查三关
01 locator 稳定性 6 项检查
✗ 反例 · 偷工减料
await page.locator('input[type="text"]').fill('alice');
await page.locator('button').click();
await page.locator('a.book-card').first().click();
✓ 正例 · user-facing
await page.getByLabel('用户名').fill('alice');
await page.getByRole('button', { name: '登录' }).click();
await page.getByRole('link', { name: '重构' }).click();
CHECKLIST · 6 项
  • 没有 CSS 长路径（nth-child / > / :nth-of-type）
  • 没有自动生成的类名（_abc123）
  • 优先 getByRole/getByLabel/getByText/getByPlaceholder
  • 少用 .first()，必须用要加注释说明
  • 没有 XPath（除非真的没语义）
  • 同组元素定位策略一致
RULE用户视角语义永远优先于 CSS 路径。例外只有第三方组件库的奇怪结构 → 用 testid，且需团队约定。
