EP 05/审查三关
03 断言深度 最难发现 · 最危险
🟢 SAFE
业务断言
toHaveText('1')
toHaveCount(3)
断完 UI 现象再断业务状态，真验证。
🟡 SHALLOW
浅断言
toBeVisible()
UI 出现 ≠ 业务对。点了按钮但没验证数据变化。
🔴 ALWAYS GREEN
永远绿
or() / toBeTruthy()
用例发现不了任何 bug。
⚠ 最危险 用例名"结账成功"，但永远绿 
await expect(
  page.getByRole('heading', { name: '下单成功' })
    .or(page.locator('.field-error'))
).toBeVisible();
下单成功 heading 出现 → 通过 ✓ · 表单校验失败 .field-error 出现也 → 通过 ✓
下单成功也绿，下单失败也绿，bug 根本测不出来。但它能跑，能进 CI，让你觉得"业务被保护了"。
RULE用例名说什么，断言就只断什么。失败用例要正反都断：错误出现 ∧ 成功页未出现。
