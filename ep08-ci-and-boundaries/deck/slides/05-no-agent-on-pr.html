EP 08/CI 与边界

为什么 PR 不跑 agent

REASON · 01
慢
Planner 探索一次 3-5 分钟，Generator 生成一次 5-8 分钟。每个 PR 等 10 分钟以上 —— 开发同学会禁用 CI。
REASON · 02
不可重复
同一个 prompt 在不同时刻 / 温度下输出不一样。CI 拿什么做 diff？
REASON · 03
会改文件
Generator 写 spec / Healer 改 spec —— CI 自动改了文件触发新的 CI，无限循环。
触发点行为
PR→ 只跑现有 spec
Nightly→ 跑全量 + 基线 diff
开发者本地 · 手动→ 跑 agent 走半自动 SOP
COREagent 是开发工具，不是 CI 工具。这一点搞错，CI 会变成测试噪音工厂。
