EP 07/完整工作流

把工作流脚本化（但不要自动化）

SHELL run-agents.sh · 核心片段 
# 半自动 SOP 引导脚本
pause() {
  yellow "→ $1"
  # 等用户回车，不是自动继续
  read -p "  完成后回车，Ctrl+C 中止：" _
}

# 每个 agent 调用前都 pause
pause "去 Claude Code 跑 Planner，
       产物 test-plan 更新好后回车"

pause "review test-plan，
       review 完回车"

pause "去 Claude Code 跑 Generator，
       specs 写完后回车"

npx playwright test
# 红的进 Healer 守门 prompt
DESIGN PRINCIPLE
每一步都 pause，等你回车才继续。
这是流程引导员，不是自动化执行器。
  • 不让 AI agent 自己唤起 AI agent —— 唤起决策权留在人手里
  • "全自动" = "全风险" —— 前面 6 集 review 机制全失效
  • 固化 SOP 5 步 —— 团队新人按脚本走，不会跳步
  • 本质：流程引导员，把"该停的地方"都标出来
⚠这不是"一键全自动"。脚本只是固化 SOP，团队按它走不会跳步 —— 这个区别请记住。
