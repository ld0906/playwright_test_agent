EP 08/CI 与边界

4 种不该用 Test Agents 的场景

01
高风险关键业务
支付、医疗、风控、合规
— 错一个边界就是钱、命、法律责任
风险：AI 生成断言倾向"看起来对"，不倾向"穷举边界"。手写 + 资深测试结对评审。
02
页面没有稳定语义
第三方组件库 SSR、Canvas 图表、iframe
— 没有 ARIA / label / testid
风险：Planner 拿不到稳定 locator，Generator 写 CSS 长路径 —— 回到录制脚本老路。先治可访问性。
03
缺少 seed 文档
祖传系统没 PRD、产品拒绝写规则
— 需求只在群聊和会议里
风险：Planner 在 UI 表面打转，发现不了业务 bug。可用 Planner 倒逼业务方梳理。
04
团队没 review 文化
PR 秒批、测试代码没人看
— 断言写啥都过
风险：AI 偷工减料直接进 CI、Healer 灾难性修复直接放生 —— 放大团队 review 缺失。
自查门槛这 4 条全要过才用 Test Agents。任何一项不过 —— 先解决那项，再来。
