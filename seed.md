## 1. 应用是什么

**mini-bookstore** 是一个最小可控的图书购买演示站，5 个核心页面：

- `/login.html` 登录
- `/books.html` 书籍列表（含分类筛选）
- `/book.html?id={id}` 书籍详情
- `/cart.html` 购物车
- `/checkout.html` 结账

被测地址：`http://localhost:5173`

## 3. 业务规则（重要）

### 3.1 登录

- 用户名密码任一错误 → 必须显示明确错误提示，**不能跳转到列表页**
- 登录成功 → 跳转 `/books.html`
- 未登录访问任何业务页 → 自动跳回 `/login.html`

### 3.2 书籍

- 书有三个分类：`engineering` / `fiction` / `psychology`
- 每本书有 `stock`（库存）和 `discontinued`（是否下架）两个字段
- **业务规则**：库存为 0 或已下架的书，**不应能被加入购物车**
- 详情页对已下架书会显示"⚠️ 已下架"徽章

### 3.3 购物车

- 同一本书重复加入 → 数量累加（不是新增一行）
- 数量减到 0 → 自动从购物车移除
- 购物车为空 → 结账页应阻止进入或给出提示

### 3.4 结账

- 必填项：收件人、邮箱、地址、支付方式
- 邮箱必须符合 `xxx@xxx.xxx` 格式
- 姓名至少 2 字，地址至少 5 字
- 校验失败 → 在对应字段下方显示具体错误，**不应跳转到成功页**
- 校验通过 → 显示订单号，清空购物车

## 4. 风险关注点（请你重点覆盖）

按风险从高到低：

1. **越界风险**：未登录访问业务页、错误账号绕过、购物车空时进入结账
2. **业务规则误判**：已下架书 / 库存为 0 的书被加入购物车、数量异常
3. **校验绕过**：结账表单字段未填或格式错绕过校验
4. **状态不一致**：加车后购物车数字未更新、结账成功后购物车未清空

## 5. 不要测什么

避免浪费时间在这些上面：

- 性能、并发、安全攻击（不在本案例范围）
- 视觉细节（颜色、像素、间距）
- 浏览器兼容性（只关心 chromium 行为）
