EP 08/CI 与边界

CI 接入的 3 个原则

RULE · 01
PR 上跑测试
不跑 agent
agent 留给 nightly。PR 时 agent 跑得慢、不可重复、还会改文件 —— 三条都犯。
违反代价：开发同学等不及，禁用 CI 或 force push。
RULE · 02
预期失败
不算红
RISK-* 用 test.fail() 标记的进基线。CI 看到预期失败就放过，不卡 PR 门禁。
违反代价：每个 PR 都被基线 bug 卡住，CI 信号噪音化。
RULE · 03
真红卡死
不允许 retry 掩盖
retry 最多 1 次。超过就是真问题，先停下来修。
违反代价：flaky 测试也能过，红色变噪音，CI 失去信任。
⚠这 3 条任何一条破了，团队会逐渐不信任 CI，最后 CI 就成了摆设。
